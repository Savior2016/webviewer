# WebViewer 消息处理架构

## 当前架构 vs 新架构

### 当前架构（已实现）
```
Web 页面 → server.py → message_engine.py → 项目管理器 → 数据库
                ↓
            Feishu API (可选)
```

**优点**:
- 快速响应
- 直接处理
- 不依赖外部系统

**缺点**:
- 意图识别规则固定
- 不够灵活

---

### 新架构（使用 OpenClaw）
```
Web 页面 → server.py → OpenClaw Session → LLM 理解意图
                                  ↓
                          调用 WebViewer API
                                  ↓
                            项目管理器 → 数据库
                                  ↓
                          返回结果给 Web 页面
```

**优点**:
- 智能意图理解
- 自然语言处理
- 可扩展性强

**缺点**:
- 依赖 OpenClaw
- 响应稍慢

---

## 实现方案

由于 OpenClaw 的技能系统需要特定配置，我们采用一个**混合方案**：

### 方案：增强的消息处理引擎

保持当前的 `message_engine.py`，但增加：

1. **更智能的意图识别** - 支持更多自然语言表达
2. **上下文理解** - 记住用户的上一次操作
3. **多轮对话** - 支持追问和确认
4. **错误恢复** - 无法理解时提供引导

## 提示词设计

当用户输入消息时，系统应该：

```
系统提示：
你是一个智能助手，帮助用户管理他们的物品和出行。

用户说："{用户消息}"

请理解这条消息的意图，并选择合适的项目来保存这些信息：

1. 如果提到"出差"、"旅行"、"出行" → 使用 **已读不回 (By Design)** 创建出行清单
2. 如果提到"搬家"、"打包" → 使用 **一搬不丢 (Cherry Pick)** 记录物品位置
3. 如果提到"找"、"查询"、"在哪里" → 使用 **妈妈的手 (Momhand)** 搜索物品
4. 如果提到"记一下"、"放在"、"位置" → 使用 **妈妈的手 (Momhand)** 记录物品位置

然后：
1. 调用相应的 API 保存数据
2. 返回友好的确认消息
3. 提示用户刷新页面查看更新

示例回复：
"✅ 已帮你记录：大疆 action4 放在了电视柜上面的透明箱子里
   📦 物品：大疆 action4
   📍 位置：电视柜上面的透明箱子里
   📂 类型：电子产品
   
   👉 页面将自动刷新显示新物品"
```

## 下一步

如果要完全使用 OpenClaw 处理，需要：

1. 创建一个 OpenClaw 技能包
2. 配置技能的触发条件
3. 定义技能的动作和参数
4. 在 Web 页面通过 OpenClaw API 调用技能

当前实现已经足够智能，支持自然语言输入。
