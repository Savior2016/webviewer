## 1. 项目概述

### 1.1 项目简介

WebViewer 是一个**集成式智能工具箱平台**，集成了四个独立的生活管理工具，通过统一的聊天界面实现自然语言交互。系统采用 Web 架构，支持 HTTPS 加密访问，配备完整的访问控制和审计功能。

### 1.2 核心价值

| 价值维度 | 说明 |
|----------|------|
| **一站式管理** | 四个工具集成在一个平台，统一入口 |
| **自然语言交互** | 通过聊天窗口用自然语言操作所有工具 |
| **安全可控** | 访客访问需审批，完整审计日志 |
| **零依赖部署** | 基于 Python 标准库，无需额外依赖 |

### 1.3 用户场景

- ✈️ **出行规划** - 快速创建出差/旅行清单，自动加载检查项
- 📦 **搬家管理** - 记录物品打包位置，追踪搬家进度
- 🏠 **物品管理** - 记录物品存放位置，快速查找
- 🤖 **智能助手** - 自然语言对话，自动识别意图并执行操作

## 5. 统一聊天功能

### 5.1 聊天组件架构

聊天组件 (`/www/js/agent-chat.js`) 是工具箱的核心交互界面：

```javascript
// 核心流程
sendMessage() → POST /api/send-message → 消息处理引擎
                                           ↓
pollResult() ← GET /api/message-result ← 保存结果
```

### 5.2 消息处理引擎

**文件**: `/root/.openclaw/workspace/message_engine.py`

**处理流程**:
1. **意图解析** (`parse_intent`) - 根据关键词识别用户意图
2. **项目路由** - 路由到对应项目管理器
3. **执行操作** - 调用管理器的具体方法
4. **结果返回** - 格式化返回结果

**意图识别规则**:

| 关键词 | 项目 | 动作 |
|--------|------|------|
| 出差、旅行、出行、旅游 | By Design | create_trip |
| 搬家、打包、物品记录 | Cherry Pick | add_item/create_move |
| 找、查询、在哪里 | Momhand | search_item |
| 记一下、记录、放在 | Momhand | add_item |

### 5.3 API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/send-message` | POST | 发送消息 |
| `/api/message-result` | GET | 获取处理结果 |
| `/api/settings` | PUT | 保存设置 |

## 6. 登录与权限管理

### 6.1 双模式访问控制

#### 管理员模式
- 🔐 **账号密码登录** - SHA256 加密存储
- 🎫 **会话管理** - 12 小时有效会话
- 📊 **完整权限** - 访问所有功能和审计后台

#### 访客模式
- 🚀 **访客申请** - 一键提交访问申请
- ⏳ **等待审批** - 实时轮询审批状态
- 🕐 **临时会话** - 审批通过后 24 小时有效

### 6.2 访问审批流程

```
┌─────────┐         ┌─────────┐         ┌─────────┐
│ 访问者   │         │  系统   │         │ 管理员   │
└────┬────┘         └────┬────┘         └────┬────┘
     │                   │                   │
     │ 1. 访问网站       │                   │
     │ ─────────────────>│                   │
     │                   │                   │
     │                   │ 2. 飞书通知       │
     │                   │ ─────────────────>│
     │                   │                   │
     │ 3. 显示等待页面   │                   │ 4. 点击审批
     │ <─────────────────│                   │
     │                   │                   │
     │ 5. 轮询状态       │                   │
     │ ─────────────────>│                   │
     │                   │                   │
     │ 6. 审批通过       │                   │
     │ <─────────────────│                   │
     │                   │                   │
```

### 6.3 审计日志系统

**记录内容**:
- 🌐 IP 地址
- 🖥️ 设备信息（浏览器、操作系统）
- 📍 访问路径
- ⏰ 时间戳
- 📊 访问状态

**日志查询**:
- 多维度筛选（IP、浏览器、路径）
- 统计面板（总访问数、独立 IP、今日访问）
- 分页加载（每次 50 条）

### 6.4 安全特性

| 特性 | 说明 |
|------|------|
| **HTTPS 加密** | 所有通信使用 SSL/TLS |
| **会话令牌** | 唯一会话 ID，防止伪造 |
| **访问日志** | 完整记录所有访问尝试 |
| **白名单机制** | 可信 IP 直接放行 |
| **会话过期** | 管理员 12 小时，访客 24 小时 |

- Python 3.11+
- Linux/Unix 系统
- 443 端口权限（或使用反向代理）

### 9.2 启动流程

```bash
# 1. 设置环境变量
export FEISHU_APP_ID="cli_xxxxx"
export FEISHU_APP_SECRET="xxxxx"
export FEISHU_USER_OPEN_ID="ou_xxxxx"

# 2. 检查配置
python3 check_config.py

# 3. 启动服务
./start.sh

# 4. 验证
ps aux | grep server_enhanced
curl -k https://localhost:443
```

### 9.3 管理命令

```bash
# 查看服务状态
ps aux | grep server_enhanced

# 查看实时日志
tail -f server.log

# 重启服务
pkill -f server_enhanced.py
cd /root/.openclaw/workspace/webviewer && nohup python3 server_enhanced.py > server.log 2>&1 &

# 查看统计信息
python3 -c "from server_enhanced import data_manager; print(data_manager.get_stats())"
```

### 10.2 故障排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 服务无法启动 | 端口被占用 | `lsof -i :443` 检查并释放端口 |
| 飞书通知失败 | Token 过期 | 重新获取飞书 App Token |
| 聊天功能超时 | AI 响应慢 | 检查 OpenClaw Agent 状态 |
| 审计日志不记录 | 权限问题 | 检查 `data/` 目录写权限 |
