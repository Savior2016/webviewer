# WebViewer 对话系统 - 完整实现

## 🎯 系统设计

### 架构说明

```
用户在主页面对话框输入消息
         ↓
POST /api/send-message
         ↓
message_engine.py 解析意图
    - 理解自然语言
    - 识别项目类型
    - 提取关键信息
         ↓
调用对应项目管理器
    - By Design (出行)
    - Cherry Pick (搬家)
    - Momhand (物品)
         ↓
保存数据 + 返回结果
         ↓
对话界面显示结果
         ↓
3 秒后自动跳转到相关页面
```

1. **聊天头部**

2. **消息区域**
   - 用户消息（右侧，蓝色）
   - 助手消息（左侧，白色）
   - 自动滚动

3. **输入区域**
   - 大尺寸输入框
   - 发送按钮
   - 快捷示例按钮

### 交互流程

```
1. 用户输入："帮我记一下，我的大疆 action4 放在了电视柜上面的透明箱子里"
         ↓
2. 点击发送 / 按 Enter
         ↓
3. 显示用户消息（蓝色气泡）
         ↓
4. 显示"输入中..."动画
         ↓
5. 调用 API 处理
         ↓
6. 显示助手回复（白色气泡，带 emoji）
         ↓
7. 3 秒后自动跳转到 Momhand 页面
```

## 🎨 提示词设计

### 系统提示（隐含在 message_engine.py 中）

```
你是一个智能助手，帮助用户管理他们的物品和出行。

用户说："{用户消息}"

请理解这条消息的意图，并选择合适的项目来保存这些信息：

1. 如果提到"出差"、"旅行"、"出行" → 使用 **已读不回 (By Design)** 创建出行清单
2. 如果提到"搬家"、"打包" → 使用 **一搬不丢 (Cherry Pick)** 记录物品位置
3. 如果提到"找"、"查询"、"在哪里" → 使用 **妈妈的手 (Momhand)** 搜索物品
4. 如果提到"记一下"、"放在"、"位置" → 使用 **妈妈的手 (Momhand)** 记录物品位置

然后：
1. 调用相应的 API 保存数据
2. 返回友好的确认消息
3. 提示用户刷新页面查看更新
```
